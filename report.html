Project 3 - Transfer Learning¶

In this project I used the driving dataset provided by Udacity to train a model using a convolutional neural network. The only inputs to the model are the steering angles and camera images from the left, center, and right cameras mounted on the vehicle driving in a simulated environment.

The model is trained to predict the steering angle based on the vehicles location on the track and other features from the camera images.

Data Preprocessing and Creating Additional Data¶

I ended up only using the Udacity dataset because I found it difficult to record clean data that wouldn't make model train the car irratically.

The simulator records data into a csv log file which references left, center, and right images. The log file also records other telemetry data including steering, throttle, brake, and speed. However, I only used steering in this project.

My first preprocessing step was to resize the original images to a smaller footprint. This parameter seemed to have quite a large impact on the overall performance of model. I tried numerious sizes and the one that seemed to work the best was 16 Height by 32 Width.

I decided to convert the original RGB color images into HSV color space. I then chose to just use the 'S' or saturation portion because this seemed to easly differentiate between the road and the other obstacles. By doing this it also drastically reduced the size of the data that had to be processed which would make training much quicker. I also tried converting the original color images into grayscale images but this didn't workout as well as the HSV images.

I also decided to add more data to the Udacity dataset by flipping the images about the y-axis and reversing the steering angle direction. This would make sure that the model wouldn't be biased toward making more left turns.

Finally, I decided to also artificially modify the steering angles of the left and right camera images by increasing their angles. This was done in an attempt to bring the car back to the center of the track. I tried a few different values for this but I got the best results by adding/subtracting 0.4 degrees to each cooresponding turn.

Development of Model Architecture¶

I first tried to train my model using Nvidia's and Commaai's CNNs. These models seemed like they should have provided good results since these models are already being used to drive by cloning the driving behavior of cars. I tried using each of these a few times but they both take a long time to train due to the large number of parameters in each. Instead I decided to start with a very low level network and build onto it if needed.

The input images were first preprocessed in order to cut down on the training time and also to reduce the degrees of freedom for the network to get lost in.

The model I ended up using is very basic in design and I am suprised that it worked as well as it did. The preprocessing, model creation, and model training was coded in python using keras (tensorflow) and is saved as model.py.

Input¶

The input is 16x32x1 with 1 for S of HSV color channels.

Model Architecture¶

  • Layer 1: Normalization to range -1, 0

  • Layer 2: Convolution with strides=(2,2), valid padding, kernel 2x2 and output shape 14x30x2, with relu activation

  • Layer 3: Maxpooling with strides (4,4), kernel 4x4, valid padding, Dropout of 30%, and output shape 3x7x2

  • Layer4: Flatten with output shape 42

Output¶

Layer Fully Connected with 1 output value for the steering angle.

Training the Model¶

I used the original data provided by Udacity and I increased the size of this dataset by flipping the original images/steering angle and adding them to the original dataset.

I tried to create additional data by using the simulator in training mode but I felt like the steering input were not very smooth and I thought this might adversely affect the training of the model.

I validated the training set by setting 10% of the total training data aside for validation.

The Adam Optimizer was used with a mean squared error loss. The batch size of 128 and 10 epochs were initialized at the beginning of model.py. The model trained on 43,389 randomized features and validated the results on 4,821 random sample images.

In order to pass the assigment the vehicle must drive around the first track staying on the road and not going up on or beyond the curb.

The trained model, (saved as model.json with weights saved as model.h5), is used in the simulator to test the results. The simulator outputs the center camera image, along with steering and throttle telemetry. The trained model takes those inputs and outputs new steering angle and throttle values. The throttle was hard coded to 0.2 and the input image was resized, the same as for training.

I tested the model on both track 1 and 2. The car swerved back and forth some of the time but they performed very well and did not go up on any curbs.
